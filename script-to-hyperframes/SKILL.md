# 脚本转HyperFrames

将结构化的视频脚本转换为详细的HyperFrames合成提示词。

## 工作流程

### 第一步：读取脚本

阅读视频脚本文件，理解：

- **基本信息**：内容类型、时长、目标受众、核心信息
- **分镜脚本**：时间点、画面描述、台词/字幕、音频提示
- **样式要求**：强调关键词、颜色、字体
- **制作提示**：格式、风格、布局、技术规格

### 第二步：提取关键元素

提取并结构化以下内容：

1. **目标受众与平台** - 用于发现阶段
2. **场景结构** - 场景数量、时间点、转场
3. **视觉风格** - 情绪、色彩、排版（基于脚本提示）
4. **动画需求** - 文字效果、转场、场景切换
5. **音频需求** - BGM风格、配音、音效

### 第三步：生成HyperFrames提示词

使用 [PROMPT_TEMPLATE.md](references/PROMPT_TEMPLATE.md) 中的模板结构化提示词，包括：

- **设计系统要求** - 颜色、排版、情绪
- **合成结构** - 场景分解、时间点、布局
- **动画规范** - GSAP模式、转场、效果
- **技术要求** - 格式、尺寸、输出规格

### 第四步：验证输出

确保生成的提示词：

- 遵循HyperFrames命名规范（data-*、GSAP模式）
- 包含所有不可协商规则（确定性、时间线契约等）
- 指定场景之间的转场（无跳跃剪辑）
- 为每个元素定义进场动画
- 包含设计系统细节或引用design.md

## 脚本格式支持

此技能支持包含以下部分的脚本（中文或英文）：

- `基本信息` / `Basic Info` - 内容类型、时长、受众、核心信息
- `分镜脚本` / `Scene Breakdown` - 时间、画面、台词、音频
- `字幕重点` / `Subtitle Highlights` - 关键词、强调、样式
- `制作提示` / `Production Notes` - 格式、风格、技术规格

对于没有完整结构的脚本，提取可用内容并使用HyperFrames house-style默认值。

## 关键映射

| 脚本元素 | HyperFrames概念 |
|---------|----------------|
| 时间 (0-3s) | `data-start`、`data-duration` |
| 画面 | 场景布局、CSS样式 |
| 台词/字幕 | 文本内容、字幕样式 |
| 音效/备注 | 音频轨道、转场时间 |
| 文字动效 | GSAP动画 (from/to) |
| 转场 | 转场模式（淡入淡出、擦除、着色器） |
| BGM风格 | 情绪选择，如需要则音频响应 |

## 输出示例

将脚本转换为包含以下内容的结构化提示词：

```
## 设计系统
- 目标受众：[从脚本提取]
- 情绪：[根据BGM风格和内容推断]
- 色彩方案：[品牌色或根据情绪推荐]

## 场景结构
1. [开场] (0-3s)：[描述]
2. [痛点] (3-8s)：[描述]
...

## 动画规范
- 文字进场：gsap.from() 模式
- 转场：[类型]
- 节奏：[快速/中速/慢速]
```
